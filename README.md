# china-tianbosports-pages

本仓库用于归档和发布多个独立 HTML 页面。

## 项目简介

这是一个以静态页面归档为主的仓库，主要用于存放各类独立 HTML 文件。每个页面均可通过 GitHub Pages 直接访问，适合用于展示、测试、存档或作为内容分发的基础单元。

仓库不针对任何特定网站或域名，仅作为页面文件的集合载体。

## 目录说明

```
.
├── index.html          # 仓库首页 / 导航索引页面
├── pages/              # 独立 HTML 页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源文件（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `index.html`：可选的起始页面，用于展示页面列表或导航。
- `pages/`：主要页面存放位置，每个文件为一个独立完整的 HTML 页面。
- `assets/`：多个页面共享的样式、脚本或图片资源。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖外部数据库或后端服务。
- 页面之间相互独立，修改或添加某个页面不会影响其他页面。
- 归档内容可能包含静态展示页、工具页、信息页等各类用途的页面。
- 页面命名建议使用英文小写、连字符分隔，便于 URL 访问。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 提交并推送至 GitHub，开启 GitHub Pages 后即可通过仓库 Pages 链接访问对应页面。

如需添加新页面，建议先在本地测试无误后再提交。
